# 🏫 우리학교 생활 달력

기존 **우리학교 학사일정 캘린더 3.2 안정판**을 건드리지 않고 새 레포지토리에서 시작하는 확장판입니다.

학사일정, 급식, 시간표를 한 화면에서 확인하는 것을 목표로 합니다.

---

## 현재 버전

**v0.3 학사일정 달력 표시**

- 기존 3.2 코드와 별도 프로젝트로 분리
- 학교 검색 결과 카드 표시
- 학교 선택 시 상단바 학교명 반영
- 선택 학교 localStorage 저장
- 학교변경/다시 선택 흐름 정리
- 월별 달력 화면 구성
- 학사일정 API 연결
- 일정이 있는 날짜에 달력 배지 표시
- 날짜 클릭 시 해당 날짜 학사일정 상세 표시
- 월 이동 시 해당 월 학사일정 재조회
- 급식·시간표는 다음 단계 연결 예정
- Render/Vercel 프록시 서버 예시 포함

---

## 화면 구성

### 1. 학교 찾기

지역과 학교명을 입력해 학교를 검색합니다.

### 2. 월별 달력

선택한 학교의 월별 정보를 달력으로 표시합니다.

달력에는 먼저 학사일정을 표시합니다.

- 📅 일정
- 📅 시험
- 📅 행사
- 📅 방학/휴업

급식과 시간표 표시는 다음 버전에서 순차적으로 연결합니다.

### 3. 날짜별 상세 보기

날짜를 누르면 아래에서 상세 내용을 확인합니다.

- 학사일정
- 급식 메뉴: v0.4 연결 예정
- 학년·반별 시간표: v0.5 이후 연결 예정

## API 연결 구조

GitHub Pages에는 API 키를 직접 넣지 않습니다.

```txt
GitHub Pages 프론트엔드
        ↓
Render / Vercel 프록시 서버
        ↓
NEIS Open API
```

---

## 사용 예정 NEIS API

### 학교 검색

- `schoolInfo`
- 학교명, 교육청 코드, 학교 코드, 학교급, 주소 조회

### 학사일정

- `SchoolSchedule`
- 월별 학사일정 조회

### 급식

- `mealServiceDietInfo`
- 날짜별 급식 메뉴, 칼로리, 영양정보 조회

### 시간표

학교급에 따라 API가 다릅니다.

- 초등학교: `elsTimetable`
- 중학교: `misTimetable`
- 고등학교: `hisTimetable`
- 특수학교: `spsTimetable`

## 배포 방법

### 1. 프론트엔드

새 GitHub 레포지토리를 만들고 아래 파일을 업로드합니다.

- `index.html`
- `style.css`
- `app.js`
- `README.md`

GitHub Pages를 켜면 프론트 화면이 열립니다.

### 2. 프록시 서버

`server` 폴더를 Render 또는 Vercel에 배포합니다.

환경변수에 아래 값을 넣습니다.

```txt
NEIS_API_KEY=나이스_API_인증키
```

### 3. 프론트에서 프록시 주소 변경

`app.js` 상단의 주소를 실제 배포 주소로 바꿉니다.

```js
const API_CONFIG = {
  useMockOnError: true,
  baseUrl: "https://배포한-프록시-주소"
};
```

## 주의사항

- 기존 `school-calendar` 3.2 안정판은 수정하지 않습니다.
- 이 프로젝트는 새 레포지토리에서 시작하는 확장판입니다.
- 학사일정은 프록시 서버와 NEIS API 키가 정상 연결되어야 실제 데이터가 표시됩니다.
- 프록시 서버가 연결되지 않으면 일부 예시 데이터가 표시될 수 있습니다.
- 시간표는 학교급, 학년도, 학기, 학년, 반, 날짜가 맞아야 조회됩니다.
- 학교에 따라 시간표 또는 급식 정보가 제공되지 않을 수 있습니다.
